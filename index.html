Cистеми контролю версій та Git
Git — це популярна розподілена система контролю версій, яка дозволяє кожному розробнику мати повну копію репозиторію. Основні команди Git включають git init (створення репозиторію), git add (додавання змін), git commit (збереження змін), git push (відправка змін на віддалений сервер) і git pull (отримання змін з сервера).
Системи контролю версій (VCS) — це програмні інструменти, що допомагають керувати змінами в коді та інших файлах проектів. Вони зберігають історію змін, дозволяють співпрацювати в команді, відстежувати версії, а також повертатися до попередніх станів проєкту. Існують централізовані (як Subversion) та розподілені системи (як Git).
Основні можливості та переваги використання Git

Можливості:

Локальна робота з репозиторієм:

Кожен користувач має локальну копію всього репозиторію, що дозволяє працювати з кодом без постійного доступу до інтернету.

Відстеження змін:

Git фіксує всі зміни, зроблені в проєкті, дозволяючи повернутися до попередніх версій файлів.

Гілки (branches):

Можна створювати незалежні гілки для роботи над різними функціями або виправленням помилок, без впливу на основну версію коду.

Злиття змін (merge):

Після завершення роботи в гілці її зміни можуть бути легко об'єднані з основною версією коду.

Віддалені репозиторії:

Можна синхронізувати локальний репозиторій з віддаленими серверами (наприклад, на GitHub або GitLab).

Резерв конфліктів:

Git пропонує інструменти для вирішення конфліктів, що виникають під час злиття змін.

Історія комітів:

Можна переглядати детальну історію змін, включаючи коментарі до комітів, авторів і час внесення змін.

Переваги:

Розподіленість:

Кожен користувач має повну копію репозиторію, що підвищує стійкість до втрати даних і дозволяє працювати офлайн.

Гнучкість роботи з гілками:

Легка робота з гілками дозволяє ефективно реалізовувати нові функції, експериментувати та виправляти помилки без ризику зламати основну версію коду.

Швидкодія:

Більшість операцій виконується локально, тому Git працює дуже швидко навіть з великими проєктами.

Відкритий код та підтримка спільноти:

Git є відкритим програмним забезпеченням з великою спільнотою, що забезпечує доступ до навчальних матеріалів, інструментів та підтримки.

Безпека:

Історія комітів захищена від модифікацій, що гарантує прозорість і цілісність змін.
10 базових команд роботи в системі Git з коментарями

1. git init

Ініціалізує новий Git-репозиторій у поточній директорії. Створює приховану папку .git, де зберігаються всі версії та коміти проекту.
git init

2. git clone < url >

Клонує (завантажує) існуючий репозиторій з віддаленого серверу на локальний комп'ютер.
git clone https://github.com/user/repository.git

3. git add < file >

Додає зміни файлу або файлів до індексу (стейджингу) для підготовки до коміту.
git add file.txt

4. git commit -m "коментар"

Фіксує зміни в локальному репозиторії з коментарем, що пояснює зміни.
git commit -m "Додано нову функцію"

5. git status

Показує поточний стан робочого каталогу, включаючи інформацію про файли, які були змінені, додані або готові для коміту.
git status

6. git pull

Завантажує та об'єднує зміни з віддаленого репозиторію у вашу локальну гілку.
git pull origin main

7. git push

Відправляє зміни з локального репозиторію на віддалений репозиторій.
git push origin main

8. git branch

Показує список існуючих гілок у репозиторії або створює нову гілку.
Показати всі гілки:
git branch
Створити нову гілку:
git branch new-feature

9. git checkout < branch >

Перемикає робочу директорію на вказану гілку або коміт.
git checkout main

10. git merge < branch >

Об'єднує зміни з вказаної гілки до поточної гілки.
git merge new-feature
